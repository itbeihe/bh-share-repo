WHY CSS Reset

  • 浏览器默认样式的不一致
  • 浏览器默认样式反人类
  • 项目中设计规范的通用样式

浏览器默认样式不一致

浏览器/标签 p
ie6 display: block;
ie7 display: block;
ie8 display: block; margin: 1em 0;
ie9 display: block; margin: 1em 0;
Firefox 3.6 display: block; margin: 1em 0;
WebKit 61376 display: block; margin: 1.0__qem 0px;

浏览器默认样式反人类

浏览器/标签 BODY
ie6 display: block; margin: 15px 10px; zoom: 1;
ie7 display: block; margin: 15px 10px; zoom: 1;
ie8 display: block; margin: 8px; zoom: 1;
ie9 display: block; margin: 8px; zoom: 1;
Firefox 3.6 display: block; margin: 8px;
WebKit 61376 display: block; margin: 8px;

CSS reset 起始

据玉伯的叙述，第一个CSS reset是Tantek写的undohtml.css。

粗暴的CSS reset

*{ margin:0; padding:0; }

错杀一千，不放过一个。偷懒时候常用。

滥用的CSS reset

blockquote,body,button,dd,dl,dt,fieldset,form,h1,h2,h3,h4,h5,h6,hr,input,legend,li,ol,p,pre,td,textarea,th,ul{margin:0;padding:0}
  • body只要margin:0;就over了
  • p标签只需要margin:0;或margin:1em 0;
